TEACHER

<p><strong>Introduction to creating exercises</strong></p>
<p>Every exercise you as a teacher will be creating needs the same basic parameters: a title, a language and the information whether the text shown to students should be containing upper- and lowercase letters as well as blanks. Additionally you always need to provide a plaintext. The program will then - if necessary - automatically display the ciphertext. For certain exercises more information is required.</p>
<p>In the toolbar you have several "teacher" options. You can create a new exercise, revert all current changes to the previously saved version, save the exercise (this will overwrite the current version), save the exercise under a new name or delete the current exercise. Some buttons are only active when you can actually perform the corresponding action (e.g. if there are no changes you cannot click on the revert button).</p>
<p><strong>Information about Caesar 1</strong></p>
<p>In this exercise students will have to encrypt a given text manually with the help of the Caesar wheel component. For this reason it is recommended that the entered plaintext is rather short as the task may otherwise become very repetitive for students.</p>

<p><strong>Einführung: Erstellen von Übungen</strong></p>
<p>Jede Übung, die Sie als Lehrer erstellen, benötigt die selben Basis-Parameter: einen Titel, eine Sprache und die Information, ob der Test, der den Schülern gezeigt wird, Gross- und Kleinbuchstaben sowie Leerzeichen enthalten soll. Zusätzlich muss jeweils ein Klartext eingegeben werden. Das Program wird dann - falls es gebraucht wird - automatisch den Geheimtext berechnen. Für gewisse Übungen werden noch weitere Informationen benötigt.</p>
<p>In der Toolbar gibt es verschiedene "Lehrer" Optionen. Sie können eine neue Übung erstellen, alle bisherigen Änderungen rückgängig machen bis zu der vorherigen gespeicherten Version, die Übung speichern (dies wird die aktuelle Version überschreiben), die Übung unter einem neuen Namen speichern oder die Übung löschen. Gewisse Knöpfe sind nur dann aktiv, wenn sie auch wirklich aktiviert sind (z.Bsp. wenn es keine Änderungen hat, kann man nicht auf den rückgangig machen Knopf klicken).</p>
<p><strong>Information zu Caesar 1</strong></p>
<p>In dieser Übung müssen Schüler einen gegebenen Test mit Hilfe des Caesar Rades von Hand verschlüsseln. Aus diesem Grund ist es ratsam, den Klartext eher kurz zu halten, da die Aufgabe sonst zu repetitiv wird.</p>

Dieses letzte Szenario ist ein Caesar Angriff. Um zu vermeiden, dass Schüler einen Brute-Force-Angriff durchführen, haben sie nur drei Leben, um die Aufgabe zu meistern. TEACHER

<p>The only difference between this scenario and Perm scenario 3 is that students have to rely on bi- and trigram frequencies instead of single letter frequencies. To help you create a meaningful task you have the same tools available as for the previous scenario and additonally there is a bi- and a trigram analysis view.</p>
<p>Both analysis views function in the same way. We explain it for bigrams here. The bottom row shows you the union of the ten most frequent bigrams in the chosen language together and the ten most frequent bigrams in your text. If your text is really "good", less bigrams will be displayed. The bigrams in the upper row are created by encrypting the bigrams in the lower row with the current key. We then calculate the frequencies of these bigrams in your ciphertext. If the frequencies match well with the lower row, your plaintext tends to be "better".</p>

<p>Der einzige Unterschied zum Perm Szenario 3 ist, dass Schüler sich auf die Häufigkeiten der top Bi- und Trigramme verlassen müssen. Damit Sie eine sinnvolle Aufgabe erstellen können, haben sie zusätzlich zu den bekannten Hilfsmitteln noch eine Bi- und eine Trigramm-Analyseansicht.</p>
<p>Beide Analyseansichten funktionieren auf dieselbe Art. Wir erklären es für die Bigram Analyse. Die untere Reihe zeigt die Schnittmenge der zehn häufigsten Bigramme der ausgewählten Sprache und der zehn häufigsten Bigramme des eingegebenen Klartextes. Wenn der eingegebene Klartext sehr "gut" ist, dann werden also weniger Bigramme angezeigt. Die Bigramme in der oberen Reihe werden erstellt durch Verschlüsselung der Bigramme in der unteren Reihe mit dem aktuellen Schlüssel. Wir berechnen anschliessend die Häufigkeiten dieser Bigramme in Ihrem Geheimtext. Wenn die Häufigkeiten gut passen, ist der Klartext tendentiell "besser".</p>
